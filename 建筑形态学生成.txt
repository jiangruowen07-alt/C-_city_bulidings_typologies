================================================================================
建筑形态学 / 建筑单元层 — 改动说明与生成逻辑
================================================================================
对应程序：CityLab（WinForms），与城市系统在同一 UI 内切换。

--------------------------------------------------------------------------------
一、代码与文件
--------------------------------------------------------------------------------

  BuildingModel.cs
    - 建筑场地 N×N 网格；每格一个代理，与城市「整格代理」概念一致，但语义为建筑单元。
    - 五种类型键：
        Resi 居民 | Firm 公司 | Cafe 咖啡 | Shop 商店 | P 绿地
    - 绿地 P 与城市吸引子「公园 Park」同一键，配色取 Config.AttrDefs 中 P 的色值。
    - 邻接效用 PrefNeighbor：四邻加总；建筑类之间用 Config.DefaultPrefAgent；邻格为 P
      时用 DefaultPrefAttr 中对 P 的偏好折算；P 自身为代理时有单独邻接系数表。

  CityAppForm.cs / CityAppForm.Events.cs / CityAppForm.Handlers.cs
    - 模式切换：「城市系统」/「建筑单元」；建筑侧含网格边长、图例、黑白显示按钮等。
    - ColorBuildingType(t)：四代理色 = AgentColors；绿地 = 吸引子 P 色。
    - PaintBuildingLattice：功能色绘制；_buildingBwMode 为真时仅黑（非绿地代理）/
      白（绿地），格线 #555 便于在黑色块上辨认。

--------------------------------------------------------------------------------
二、界面操作（建筑模式）
--------------------------------------------------------------------------------

  - 网格 N×N：下拉选择 3×3 … 16×16 等。
  - 图例：居民 / 公司 / 咖啡 / 商店 / 绿地(P) 与城市中配色一致。
  - 「黑白视图」按钮：关 = 五色 + 绿地绿；开 = 黑 = 除绿地外全部代理，白 = 绿地。
    导出 PNG 与当前显示一致。
  - 左键点击格子：类型在 P → Resi → Firm → Cafe → Shop → P 间循环。
  - 与城市场景共用：START/STEP/RESET/CLEAR、Run N、速度、交换规则、效用曲线与统计。

--------------------------------------------------------------------------------
三、生成 / 演变逻辑（简）
--------------------------------------------------------------------------------

  1）每步以一定概率（MutateStepChance）做「增减」：随机将一块绿地改为随机建筑类型，
     或随机将一建筑格改为绿地；否则在随机两格间做与城市场景同类的 Pareto 等交换判定。

  2）是否接受交换由当前「交换规则」与两格在换位前后的邻接效用来决定（换位时用逻辑
     上的类型映射计算「若已交换」的邻接，再与 CityModel 式规则一致）。

  3）RESET：各格在五种类型间随机；CLEAR：全格为绿地 P。

--------------------------------------------------------------------------------
四、与 README 的衔接
--------------------------------------------------------------------------------

  本文件记录「建筑形态学」实验层在代码中的落点与操作约定；主说明仍以 README.md 的
  CITY LAB 城市模拟为准。建筑层为独立 BuildingModel，不修改 CityModel 的吸引子/矩阵
  数据文件格式；配色与偏好引用现有 Config。
