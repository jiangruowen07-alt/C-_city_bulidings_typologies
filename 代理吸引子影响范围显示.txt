CITY LAB - 改动记录：代理与吸引子影响范围显示
Change Log: Agent & Attractor Influence Range Display

日期 Date: 2025-03-02

--------------------------------------------------------------------------------
一、核心算法修改 Core Algorithm Changes (model.py)
--------------------------------------------------------------------------------

1. 效用计算逻辑变更 Utility Calculation Logic
   - 原逻辑：代理按自身 reach 主动“看向”周围一定范围内的代理和吸引子
   - 新逻辑：每个代理和吸引子有影响范围；代理的满意度 = 能影响到它的源（代理+吸引子）的贡献之和

2. 新增影响范围矩阵 New Influence Range Matrices
   - influence_range_attr[k][t]: 吸引子类型 k 对代理类型 t 的影响范围（曼哈顿距离）
   - influence_range_agent[src][tgt]: 代理类型 src 对代理类型 tgt 的影响范围

3. 默认配置 Default Config
   - T(交通) 吸引子影响范围默认设为 20

4. 新增方法 New Method
   - _ensure_influence_tables_complete(): 确保影响范围矩阵完整

--------------------------------------------------------------------------------
二、UI 修改 UI Changes (ui.py)
--------------------------------------------------------------------------------

1. 偏好矩阵区域 Preference Matrix Section
   - 原 C) Agent → Reach 拆分为：
     * C) Attractor influence range (attr→agent): 行=吸引子类型，列=代理类型
     * D) Agent influence range (agent→agent): 行=源代理类型，列=目标代理类型

2. 影响范围叠加显示 Influence Overlay
   - 悬停代理：显示该代理自身的影响范围（能影响多远内的其他代理）
   - 悬停吸引子：显示该吸引子自身的影响范围（能影响多远内的代理）
   - 代理与吸引子格均可触发叠加显示

3. 单元满意度视图 Cell Satisfaction View
   - 鼠标悬停 tooltip（显示 "Utility: X.XX"）逻辑保持不变

4. 按钮与图例 Button & Legend
   - 按钮文案：SHOW INFLUENCE OVERLAY (HOVER)
   - 图例：Agent 实线=该代理对他人影响范围；Attractor 虚线=该吸引子影响范围

--------------------------------------------------------------------------------
三、C# 导出修改 C# Export Changes (export_csharp.py)
--------------------------------------------------------------------------------

1. 数据结构变更 Data Structure
   - ReachAttr, ReachAgent → InfluenceRangeAttr, InfluenceRangeAgent
   - 类型：Dictionary<string, int> → Dictionary<string, Dictionary<string, int>>

2. GetUtility 逻辑
   - 与 model.py 同步为基于影响范围的计算

--------------------------------------------------------------------------------
四、文档更新 Documentation (README.md)
--------------------------------------------------------------------------------

五、逻辑对应关系 Logic Mapping
--------------------------------------------------------------------------------

原 reach 逻辑                    新 influence 逻辑
─────────────────────────────────────────────────────────────────
reach_attr_by_type[t]            influence_range_attr[k][t]
  (代理 t 看向吸引子的范围)         (吸引子 k 对代理 t 的影响范围)

reach_agent_by_type[t]           influence_range_agent[src][tgt]
  (代理 t 看向其他代理的范围)       (代理 src 对代理 tgt 的影响范围)

代理计算：我看向周围              代理计算：谁在影响我
Agent: I look around             Agent: Who influences me
